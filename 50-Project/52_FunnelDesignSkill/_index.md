# 퍼널 디자인 스킬 프로젝트

## 프로젝트 목표
> 디지털 마케팅 퍼널 디자인 작업을 클로드 스킬즈로 만들어,
> 어떤 클라이언트와 작업하더라도 일관된 결과물을 도출할 수 있도록 한다.

---

## 접근 방식

```
[샘플 프로젝트] 모두다코리안 문서화 작업
        ↓
[패턴 추출] 반복되는 작업 흐름, 문서 구조 파악
        ↓
[스킬 제작] 재사용 가능한 클로드 스킬즈 완성
```

- 스킬즈 초안을 먼저 만들지 않음
- 실제 작업을 하면서 패턴을 발견하고 기록
- 업무가 충분히 파악된 후 스킬즈 생성

---

## 스킬 적용 대상
- **산업**: IT 서비스 유형 프로젝트
- **작업 범위**: 프로젝트 세팅, 퍼널 설계, 마케팅 문서, 운영 프로세스

---

## 작업 흐름 (발견된 패턴)

### Phase 1: 프로젝트 초기 세팅
1. 프로젝트 파악 (어떤 서비스인지, 현재 상태)
2. 폴더 구조 생성
3. 기존 문서 정리 및 이동

### Phase 2: 사업 골조 정의
1. 비즈니스 모델 정리 (수익 구조, 가격 정책)
2. 타겟 오디언스 정의 (표면/내면 구분)
3. 성장 로드맵 수립

### Phase 3: 퍼널/전략 설계
1. 전체 퍼널 구조 설계
2. 퍼널 단계별 콘텐츠 정의 (인지 → 관심 → 전환)
3. 운영 프로세스 정의 (시점별 액션)
4. 전환 스크립트 작성

### Phase 4: 마케팅 가이드
1. 메시지 프레임워크
2. 콘텐츠 전략 (Cascading 구조)
3. 채널별 제작 가이드

### Phase 5: 운영 & 개선
1. 실행 결과 피드백 기록
2. 스크립트 개선
3. 프로세스 보완

## 샘플 프로젝트: 모두다코리안

### 프로젝트 개요
- 대만 타겟 한국어 학습 서비스
- 핵심 퍼널: 말하기 챌린지 → 1:1 과외 전환
- 슬로건: "한국인은 그렇게 안 말해요. 원어민과 말하기 학습 모두다코리안"

### 핵심 인사이트: 표면 vs 내면

| 레이어 | 내용 | 마케팅 언급 |
|--------|------|-------------|
| **표면** | 말하기 학습, 발음 교정, 실생활 표현 | 직접 언급 ✅ |
| **내면** | 원어민과 대화하는 기쁨, 취미 맞는 사람과 교류, 관계 | 암시만 (직접 언급 ❌) |

### 인지 단계 콘텐츠 4유형

| 유형 | 역할 | 비율 |
|------|------|------|
| 신조어/유행어 | 가치 제공, 전문성 | 40% |
| 문제점-솔루션 | What 해결, 포지셔닝 | 20% |
| 브랜드 철학 | Why 존재, 감정 연결 | 10% |
| 주요 콘텐츠 소식 | 공감, 트렌드 | 30% |

## 스킬즈 제작 메모

> 아래는 작업하면서 발견한 인사이트를 기록하는 공간입니다.
> 나중에 스킬즈 제작시 참고합니다.

### 발견한 패턴

#### 문서 구조
- 프로젝트 초기에 기존 문서 파악 → 새 구조로 재배치하는 흐름이 있음
- 표 형식이 프로세스 정리에 효과적 (번호, 선행노드, 시점, 활동명, 목표)
- 폴더별 _index.md가 네비게이션에 유용
- **Cascading 문서 구조**: 최상위(큰 틀) → 채널별 → 콘텐츠 유형별로 계층화하면 스킬즈 Progressive Disclosure와 일치
- 백링크([[]])로 문서 간 연결하면 타고 들어가며 필요한 깊이만 로드 가능

#### 콘텐츠 분석/제작
- **기존 콘텐츠 분석 → 패턴 추출 → 문서화** 흐름이 마케팅 가이드 작성에 효과적
- 실제 성과 데이터(조회수, 좋아요)가 있으면 "무엇이 효과적인가" 판단 근거가 됨
- 콘텐츠 구조 템플릿화 (신조어 교육 패턴 등)가 일관성 유지에 중요

#### 타겟/메시지
- **표면 vs 내면 구분**: 고객이 말하는 것(표면) vs 진짜 원하는 것(내면)
- 표면은 직접 언급, 내면은 암시만 → 이 구분이 마케팅 톤 결정에 중요
- 인터뷰/미팅 메모에서 브랜드 철학 키워드 추출 가능

#### 콘텐츠 제작 방식
- **티키타카 방식**: 작업자와 대화하며 콘텐츠 제작 (주요콘텐츠소식)
- 질문 플로우: 발굴 → 맥락 → 감상 → 디테일 → 연결 → 마무리
- 초안 → 피드백 → 수정 → 확정 루프 포함

#### 인지 단계 콘텐츠
- 4가지 유형 구분: 신조어, 문제-솔루션, 브랜드철학, 콘텐츠소식
- 각 유형별 목적과 비율 설정이 콘텐츠 믹스에 중요
- 판매 의도 없음 (인지 단계) vs 전환 유도 (관심 단계) 구분

### 개선 필요 사항
- 콘텐츠 분석시 구글시트 직접 연동 어려움 → CSV 또는 복사/붙여넣기 권장
- 마케팅 문서는 As-Is 분석 후 To-Be 개선 방향 함께 제시 필요
- 인터뷰 메모가 있으면 초기에 공유받아 브랜드 철학 추출에 활용

### 질문 템플릿 (클라이언트에게 물어볼 것)

#### 초기 세팅
- "현재 어떤 채널에서 콘텐츠를 운영하고 있나요?"
- "기존 콘텐츠 데이터(게시글, 성과)를 공유해주실 수 있나요?"
- "가장 반응이 좋았던 콘텐츠는 어떤 유형이었나요?"

#### 타겟/메시지
- "고객이 왜 이 서비스를 찾는다고 생각하세요?" (표면)
- "고객이 진짜 원하는 건 뭘까요?" (내면)
- "왜 이 서비스를 시작하셨어요?" (브랜드 철학)

#### 콘텐츠 제작 (티키타카)
- "요즘 뭐 보고 계세요?"
- "어떤 점이 재밌었어요?"
- "팔로워들한테 뭘 물어볼까요?"
