# Landing Page Creation Guide

랜딩페이지 기획 문서 작성 가이드입니다.

## 개요 테이블

모든 랜딩페이지 문서는 개요 테이블로 시작합니다:

```markdown
| 항목 | 내용 |
|------|------|
| 목적 | [전환 목표: 신청, 구매, 문의 등] |
| 타겟 | [타겟 오디언스 + 핵심 니즈] |
| 유입 경로 | [어디서 오는가: SNS, 광고, 검색 등] |
| CTA | [최종 행동: 버튼 텍스트] |
```

---

## 페이지 구조 원칙

### 고정 vs 가변 섹션

```
[가변 섹션] - 캠페인/기수별 변경
└─ 히어로, 모집 요약, 모집 상세, CTA

[고정 섹션] - 내용 변경 없음
└─ 페인포인트, 솔루션, 상세 설명, 가격, 신뢰 요소, FAQ
```

> ⚠️ **관리 포인트**: 가변 섹션만 캠페인마다 수정

### 표준 섹션 구조 (12개)

```
[상단 - Attention]
1. 히어로 섹션
2. 모집 요약 (핵심 정보 빠른 확인)

[중단 - Interest → Desire]
3. 페인포인트 (공감)
4. 솔루션 제시
5. 진행 방식/상세
6. 일정/구조
7. 가격/시스템

[하단 - Trust → Action]
8. 제공자 소개 (선생님, 전문가 등)
9. 참여자 반응/후기
10. FAQ
11. 모집 정보 상세
12. CTA
```

## 섹션별 가이드

### 1. 히어로 섹션

**목적:** 즉각적 관심 유발 + 핵심 가치 전달

**필수 요소:**
- 메인 카피 (핵심 가치 1문장)
- 서브 카피 (무엇인지 설명)
- 혜택 배너 (있을 경우)
- 비주얼 (캐릭터, 제품 이미지)
- CTA 버튼

**예시:**
```
[메인 카피]
每天開口說韓文，是提升口說實力的最好方法！

[서브 카피]
跟韓國老師每天練習、每天獲得反饋
21天韓文口說挑戰

[혜택 배너]
🎉 首次參加者 參加費全額免除！

[CTA]
立即報名 →
```

---

### 2. 모집 요약 (상단)

**목적:** 핵심 정보 빠른 확인 (스크롤 피로 감소)

**필수 요소:**
- 기간/날짜
- 인원/수량
- 가격/혜택
- 작은 CTA (↓ 자세히 보기)

**예시:**
```
📅 [시작일] ~ [종료일]
👥 限 15 名
💰 首次參加: 保證金 3000元 (參加費免除！)

↓ 了解更多
```

---

### 3. 페인포인트 섹션

**목적:** "나의 이야기"라고 느끼게 하기

**구조:**
```
[헤딩] 질문형으로 공감 유도
[리스트] 3~5개 페인포인트 (이모지 + 설명)
[전환] "이런 고민이 있다면... 👇"
```

**주의:**
- 표면 페인포인트만 언급 (스킬 향상, 문제 해결)
- 내면은 암시만 (관계, 소속감)

---

### 4. 솔루션 제시

**목적:** 이 서비스가 해결책임을 인식

**구조:**
```
[헤딩] 서비스/제품명
[핵심 가치] 3가지 (✅ 아이콘 + 한 줄 설명)
[한 줄 요약] 차별점 강조
```

---

### 5. 진행 방식/상세

**목적:** "구체적으로 뭘 하는 거지?" 해소

**구조:**
```
[헤딩] "어떻게 진행되나요?"
[스텝별 설명] 아이콘 + 제목 + 설명
```

---

### 6. 일정/구조

**목적:** 전체 구조 시각화

**구조:**
- 주차별/단계별 다이어그램
- 간단한 표 또는 타임라인

---

### 7. 가격/시스템

**목적:** 가격 설명 + 가치 증명

**필수 요소:**
- 혜택 강조 (첫 참여, 할인 등)
- 가격 구조 (정가 vs 할인가)
- 환급/보장 조건 (있을 경우)
- 강조 포인트 (💡)
- 성과 수치 (완주율, 만족도 등)

---

### 8. 제공자 소개

**목적:** 신뢰 구축

**필수 요소:**
- 사진
- 이름
- 한 줄 소개
- 자격/경력
- (선택) 캐릭터 연결

---

### 9. 참여자 반응/후기

**목적:** 사회적 증거

**구조:**
```
[헤딩] "참여자들의 이야기"
[후기] 채팅 캡처 or 텍스트 인용
[숫자] 완주율, 누적 참가자 수 등
```

---

### 10. FAQ

**목적:** 망설임 해소

**작성 원칙:**
- 실제 자주 받는 질문 5~8개
- Q: 질문 / A: 답변 형식
- 친근한 톤 유지

**필수 FAQ 주제:**
- 참가 자격/레벨
- 소요 시간
- 실패/취소 시 처리
- 환불/보장 조건
- 가격 관련

---

### 11. 모집 정보 상세 (가변)

**목적:** 구체적 일정 안내

**필수 요소:**
- 기간
- 마감일
- 모집 인원
- 비용 상세

> ⚠️ **관리 포인트**: 캠페인/기수마다 이 섹션만 변경

---

### 12. CTA 섹션

**목적:** 최종 행동 유도

**구조:**
```
[헤딩] 행동 유도 질문
[혜택 재강조]
[CTA 버튼] 크고 명확하게
[긴급성/희소성] ⚠️ 인원 한정, 마감 임박
```

## 출력 형식

### 문서 구조

```markdown
---
title: 랜딩페이지 - [제품/서비스명]
created: [날짜]
project: [프로젝트명]
tags: [marketing, landing-page, funnel]
---

# 랜딩페이지 - [제품/서비스명]

## 개요
[개요 테이블]

## 가격 정책
[가격 테이블]

## 페이지 구조
[구조 다이어그램]

## 섹션별 상세
### 1. 히어로 섹션
...

## 디자인 가이드
[톤앤매너, 컬러, 폰트]

## 수집 필요 자료
[자료 테이블 - 우선순위 포함]

## 관련 문서
[링크]
```

### 이중 언어 표기

타겟 언어와 한국어를 함께 표기:

```
每天開口說韓文
(매일 한국어로 말하기)
```

```markdown
## 디자인 가이드

- 캐릭터 활용 여부

### 컬러
- 메인: [브랜드 컬러]
- 강조: [CTA, 가격, 혜택]

- [언어별 폰트 지정]
```
