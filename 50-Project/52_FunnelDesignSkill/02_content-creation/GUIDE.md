# Content Creation Guide

작성된 가이드 문서를 기반으로 실제 콘텐츠를 제작하는 가이드입니다.

## 콘텐츠 유형별 라우팅

| 요청 유형 | 가이드 | 트리거 키워드 |
|-----------|--------|---------------|
| SNS 콘텐츠 | 이 문서 (아래 참조) | 포스팅, 콘텐츠, 피드, 스토리 |
| **랜딩페이지** | [landing-page.md](landing-page.md) | 랜딩페이지, LP, 전환 페이지 |

---

## ⚠️ 시작 전 필수: 프로젝트 문서 로드

콘텐츠 제작 전 **반드시 프로젝트 문서를 로드**해야 합니다.

### 확인 순서
1. 현재 세션에서 프로젝트 폴더가 이미 참조되었는지 확인
2. 참조된 프로젝트가 없다면 사용자에게 질문:

```
"어떤 프로젝트의 콘텐츠를 만들까요?
프로젝트 폴더 경로를 알려주세요. (예: /path/to/project)"
```

3. 프로젝트 폴더 확인 후 아래 문서들을 로드:

| 문서 | 경로 | 필요한 정보 |
|------|------|-------------|
| 타겟 오디언스 | `00_foundation/target-audience.md` | 표면/내면, Pain Point |
| 캐릭터 컨셉 | `00_foundation/character-*.md` | 캐릭터 등장 시 참조 |
| 콘텐츠 전략 | `02_marketing/content-strategy.md` | 톤앤매너, 콘텐츠 믹스 |
| 채널 전략 | `02_marketing/[채널]-strategy.md` | 채널별 가이드 |
| **콘텐츠 유형 가이드** | `02_marketing/[채널]-[유형].md` | 구체적 제작 가이드 |

4. 문서 로드 후 콘텐츠 제작 시작

---

## ⚠️ 결과물 출력 규칙

### 1. 채널별 글자수 제한

| 채널 | 글자수 제한 | 비고 |
|------|-------------|------|
| Threads | **500자 이내** | 필수 준수 |
| Instagram | 2200자 이내 | 캡션 기준 |
| 기타 | 프로젝트 문서 참조 | |

### 2. 이중 언어 출력

타겟 언어로 콘텐츠 작성 시, **한국어 버전도 함께 제공**합니다.

```
## 콘텐츠 (타겟 언어)
[타겟 언어 콘텐츠]

## 콘텐츠 (한국어 번역)
[동일 내용 한국어 버전]
```

### 3. 캐릭터 등장 시

프로젝트에 마스코트/캐릭터가 있고 콘텐츠에 등장하는 경우:
1. `00_foundation/character-*.md` 문서 확인
2. 캐릭터 성격, 말투, 톤, 호칭 참조
3. 캐릭터 체크리스트로 일관성 검토

## 콘텐츠 제작 결정 사항

### 1. 콘텐츠 유형 (무엇을)

| 유형 | 비율 | 제작 방식 | 프로젝트 가이드 |
|------|------|-----------|-----------------|
| 교육/정보성 | 40% | 직접 제작 | `[채널]-신조어교육.md` 등 |
| 트렌드/공감 | 30% | 티키타카 | `[채널]-주요콘텐츠소식.md` 등 |
| 문제-솔루션 | 20% | 직접 제작 | `[채널]-문제솔루션.md` |
| 브랜드 철학 | 10% | 직접 제작 | `[채널]-브랜드철학.md` |

### 2. 표현 형식 (어떻게)

| 형식 | 트리거 | 프로젝트 가이드 |
|------|--------|-----------------|
| **직접적** (기본) | - | 각 유형별 가이드 |
| **스토리텔링** | "스토리로", "사례로", "에피소드로" | `[채널]-스토리형.md` |

---

## 1. 직접 제작 방식

교육/정보성, 문제-솔루션, 브랜드 철학 콘텐츠에 사용합니다.

### 프로세스
```
1. 프로젝트 문서 로드
2. 콘텐츠 유형 확인
3. 해당 유형의 프로젝트 가이드 문서 참조
4. 표현 형식 확인 (직접적 vs 스토리)
5. 캐릭터 등장 여부 확인 → 캐릭터 문서 참조
6. 콘텐츠 초안 작성 (글자수 제한 준수!)
7. 타겟 언어 + 한국어 버전 함께 출력
8. 피드백 → 수정 → 확정
```

### 유형별 구조 (요약)

**교육/정보성:**
```
[핵심 정보] → [설명/배경] → [팁/예시] → [마무리]
```

**문제-솔루션:**
```
[후킹] → [공감] → [원인] → [솔루션 방향] → [마무리]
```

**브랜드 철학:**
```
[화두] → [스토리] → [깨달음] → [믿음] → [마무리]
```

**스토리텔링:**
```
[도입: 인물+상황] → [전개: 문제+시도] → [전환: 깨달음] → [결말: 변화] → [마무리: 독자 연결]
```

---

## 2. 티키타카 방식 (트렌드/공감 콘텐츠 전용)

**트렌드/공감 콘텐츠에만 사용합니다.**

작업자의 실제 반응과 경험을 대화로 끌어내어 콘텐츠에 반영하는 방식입니다.

### 왜 트렌드 콘텐츠에만?
- 트렌드 콘텐츠는 **작업자의 생생한 반응**이 핵심
- 다른 유형은 전문 지식/브랜드 스토리 기반이라 직접 제작이 효율적

### 프로세스
```
[Claude] 질문: "요즘 화제인 거 있어요?"
    ↓
[작업자] 답변
    ↓
[Claude] 후속 질문 (반복)
    ↓
[Claude] 콘텐츠 초안 (글자수 제한 + 이중 언어 + 캐릭터 체크)
    ↓
[작업자] 피드백
    ↓
[Claude] 수정 → 확정
```

→ 상세 질문 플로우: [tikitaka-flow.md](tikitaka-flow.md)

## 핵심 원칙

### 표면 vs 내면
| 레이어 | 다루는 것 | 언급 수준 |
|--------|-----------|-----------|
| 표면 | 스킬 향상, 실용적 가치 | 직접 언급 ✅ |
| 내면 | 관계, 소속감, 재미 | 암시만 (직접 언급 ❌) |

### 판매 vs 가치
- ❌ 서비스 직접 홍보 금지 (인지 단계)
- ✅ 문제 인식 + 방향 제시까지만
- ✅ 순수 가치 제공
